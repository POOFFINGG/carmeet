5. ЭКРАН «ГАРАЖ» (ГЛАВНЫЙ ЭКРАН)
5.1. Гараж — главный экран Приложения. Пользователь попадает на него при каждом
запуске (после прохождения онбординга). Это центральный экран навигации.
5.2. Элементы экрана
5.2.1. Аватарка — круглое изображение в верхней части экрана. Нажатие на аватарку
открывает галерею устройства для загрузки нового фото.
5.2.2. Ник — текст под аватаркой. Подтягивается из Telegram автоматически. Не
редактируется. Нажатие на ник ничего не делает (не является ссылкой).
5.2.3. Силуэт / AI-стилизованное изображение автомобиля — центральный элемент
экрана. Занимает основную площадь экрана. Подробности — раздел 6.
5.2.4. Кнопка настроек — иконка шестерёнки в правом верхнем углу экрана. Нажатие
открывает экран настроек (п. 5.3).
5.2.5. Фон — статичное изображение гаража. Не меняется, не анимируется. Фон должен
корректно смотреться с автомобилем любого цвета (чёрный, белый, красный, жёлтый).
Стиль: спокойный, приятный, НЕ неоновый. Конкретное изображение утверждается на
этапе дизайна.
5.2.6. Нижняя панель навигации — всегда видна (фиксированная). Подробности — раздел
12.
5.3. Экран настроек
Открывается нажатием на шестерёнку. Содержит следующие пункты:
5.3.1. Изменить автомобиль — переход на экран редактирования: марка/модель (из базы
с автодополнением), перегенерация AI-изображения (загрузка новых фото), изменение
цвета силуэта (если AI не использовался).
5.3.2. Изменить роль — переключение между Зритель / Участник / Организатор. При
переключении на Участника — предлагается добавить авто (если ещё не добавлено). При
переключении на Организатора — предлагается заполнить карточку организатора.
5.3.3. Категории интересов — экран с чекбоксами категорий/подкатегорий. Можно
изменить в любой момент. Влияет на рекомендации и уведомления.
5.3.4. Обратная связь — нажатие открывает Telegram-чат с администраторами
Приложения (через Telegram deep link). Текстового поля внутри Приложения нет.
5.3.5. НЕ входит в настройки: изменение ника (берётся из Telegram), настройки
приватности авто, система лайков/подписок.

6. AI-СТИЛИЗАЦИЯ АВТОМОБИЛЯ
6.1. Основная визуальная функция Приложения. Пользователь загружает фотографии
своего автомобиля, нейросеть генерирует стилизованное изображение в едином
мультяшном стиле.
6.2. Требования к результату генерации
6.2.1. Стиль: мультяшный / cartoon. НЕ фотореалистичный. Упрощённая графика с
сохранением узнаваемых деталей автомобиля (диски, обвесы, цвет кузова, характерные
элементы модели).
6.2.2. Ракурс: единый для всех автомобилей, статичный, вид спереди. Ракурс НЕ меняется
в зависимости от входных фотографий.
6.2.3. Формат: 2D, статичное изображение. Никакого 3D, никакого вращения, никакой
анимации.
6.2.4. Промпт нейросети: фиксированный («замороженный»), обеспечивает стабильный
единообразный стиль для всех генераций. Промпт разрабатывается и тестируется на этапе
разработки, утверждается с Заказчиком.
6.2.5. Стилистика: результат должен гармонировать с фоном главного экрана (Гараж).
Точная стилистика определяется после утверждения фона.
6.3. Процесс генерации (пользовательский сценарий)
6.3.1. Пользователь нажимает кнопку «Загрузить фото» (в онбординге или в настройках).
6.3.2. Открывается экран загрузки с пояснением: «Загрузите 4 фотографии вашего
автомобиля с разных ракурсов. Чем лучше качество фото — тем лучше результат».
Отображаются 4 слота для фотографий.
6.3.3. Пользователь выбирает 4 фото из галереи устройства. Допустимые форматы: JPG,
PNG. Минимальное разрешение: 640x480. Максимальный размер файла: 10 МБ каждое. Если
фото не соответствует требованиям — отображается сообщение об ошибке.
6.3.4. После загрузки всех 4 фото пользователь нажимает кнопку «Сгенерировать».
Отображается индикатор загрузки (спиннер или анимация ожидания) и текст «Генерируем
ваш автомобиль...». Время генерации зависит от используемого API (ориентировочно 10-30
секунд).
6.3.5. После генерации отображается результат — стилизованное изображение
автомобиля. Две кнопки: «Принять» и «Попробовать ещё раз».
6.3.6. Ограничение попыток: максимум 3 (три) попытки генерации. После исчерпания
попыток кнопка «Попробовать ещё раз» становится неактивной. Отображается текст:
«Попытки исчерпаны. Вы можете принять текущий результат или использовать силуэт.» и
кнопка «Использовать силуэт» (п. 6.5).
6.3.7. При нажатии «Принять» — изображение сохраняется в профиле пользователя и
отображается на главном экране (Гараж). Однако изображение видно ТОЛЬКО самому
владельцу (статус: «На модерации»).
6.4. Модерация AI-изображений
6.4.1. Все принятые пользователем AI-изображения попадают в очередь модерации.
Модерация осуществляется администраторами Приложения (Заказчиком) вручную.
6.4.2. Администратор видит: исходные фото пользователя, результат генерации, ник
пользователя, марку/модель авто.

6.4.3. Администратор может: «Одобрить» — изображение становится видно всем
пользователям (в списках участников мероприятий, в профиле); «Отклонить» —
пользователю отправляется уведомление через бота с текстом: «Ваше изображение не
прошло модерацию. Пожалуйста, загрузите фотографии автомобиля». Пользователю
предоставляются 3 новые попытки генерации.
6.4.4. Пока изображение на модерации — на главном экране пользователя отображается
его AI-изображение с пометкой «На модерации». Другим пользователям отображается
силуэт.
6.4.5. Интерфейс модерации — отдельная страница (веб-панель или экран внутри
Приложения, доступный только администраторам). Минимальный функционал: список на
модерации, кнопки «Одобрить» / «Отклонить» для каждого изображения.
6.5. Силуэт (при отсутствии AI-изображения)
6.5.1. Если пользователь не загрузил фото или AI-изображение не одобрено — на главном
экране отображается силуэт автомобиля.
6.5.2. Силуэт: единый для всех типов кузовов, вид спереди, тёмный контур.
6.5.3. Пользователь может выбрать цвет силуэта из палитры (8-10 цветов, определяются на
этапе дизайна).
6.5.4. Для зрителей (без авто): набор альтернативных силуэтов — велосипед, самокат,
скейт, тележка из супермаркета. Пользователь выбирает один из них.
6.6. Техническая реализация AI
6.6.1. Выбор нейросети и API определяется Исполнителем на этапе разработки.
Исполнитель проводит тестирование нескольких вариантов и выбирает оптимальный по
соотношению качество/стоимость.
6.6.2. Ежемесячные расходы на API генерации (при наличии) оплачиваются
Заказчиком отдельно и не входят в стоимость разработки. Исполнитель уведомляет
Заказчика об ориентировочной стоимости одной генерации до начала этапа разработки
AI-модуля.
6.6.3. Исполнитель обеспечивает: интеграцию с выбранным API; отправку фотографий с
фиксированным промптом; получение и сохранение результата; ограничение попыток;
отображение результата пользователю; передачу на модерацию.
6.6.4. Исполнитель НЕ гарантирует: 100% качество генерации для любого автомобиля;
идентичный результат при повторной генерации с теми же фотографиями; корректную
